// STORY MODE

Registro del proceso de desarrollo, decisiones de diseño y resolución de problemas durante el TP1.

STAGE 01 — DECISIONES DE DISEÑO

El equipo optó por una estética Arcade 90s / CRT que rompe con los diseños genéricos habituales. Se eligió una paleta de tres colores neón: cyan (#00ffff) para P1, magenta (#ff00ff) para P2 y amarillo (#ffff00) para P3, sobre un fondo negro profundo (#020408). Las tipografías Press Start 2P (pixel font para títulos) y VT323 (fuente retro para cuerpo) provienen de Google Fonts. Se incorporaron scanlines CSS animados, esquinas cortadas estilo sprite, y un ticker horizontal inferior que simula la pantalla de una máquina de arcade.

STAGE 02 — DIFICULTADES Y RESOLUCIONES

  • Desafío: Implementar los tres breakpoints obligatorios (400px, 900px, 1200px) manteniendo la estética pixel sin que los elementos se rompan en mobile.
    Solución: Se usó grid-template-columns: repeat(auto-fit, minmax(260px, 1fr)) para que las cards de jugadores se adapten automáticamente, y clamp() en el título principal para escalar la tipografía de forma fluida.
  • Desafío: Los scanlines implementados con position: fixed inicialmente generaban conflictos con el z-index de otros elementos.
    Solución: Se aplicó pointer-events: none y z-index: 9999 al pseudo-elemento ::before del body, permitiendo que los scanlines sean puramente decorativos sin interferir con la interactividad.
  • Desafío: Unificar la función JS de cambio de tema en un único archivo script-perfil.js para los tres perfiles, donde cada jugador tiene su propio color.
    Solución: Se detecta el color de borde actual de la tarjeta (getComputedStyle) para determinar el tema del jugador, y se cicla entre 5 esquemas de color que respetan el color principal del personaje.

STAGE 03 — CAMBIOS IMPORTANTES

Originalmente, se consideró usar una sola página (Single Page Application), pero se decidió seguir la consigna de archivos HTML independientes para el índice y cada integrante, facilitando así el mantenimiento y la organización del repositorio. Además, durante el desarrollo se redefinió la estética visual del sitio: se reemplazaron las tipografías y paleta de colores iniciales por una propuesta arcade con pixel fonts, colores neón y efectos CRT, para darle una identidad más distintiva al proyecto.

STAGE 04 — USO DE IA

Se utilizó Claude (Anthropic — Sonnet 4.6) como asistente técnico durante el desarrollo del TP1. La IA colaboró en la definición de la paleta de colores, el sistema de variables CSS, la implementación de los scanlines, las esquinas cortadas con clip-path, la lógica de JavaScript para el ciclo de temas y la documentación del README. Los avatares fueron generados con Gemini (Google) aplicando estilo pixel-art para preservar la privacidad de los integrantes.
